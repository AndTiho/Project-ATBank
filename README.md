# Проект "ATBank" 
## Описание
Учебный проект по созданию приложения для банка.\
**В данный момент находится в разработке, просим проявить терпение.**
## Установка:
1. Клонируйте репозиторий:
```
git@github.com:AndTiho/Project-ATBank.git
```
2. Установите зависимости:
```
poetry install
```
## Использование
1. В модуле [Маски](src\masks.py) находятся:
- Функция, которая принимает на вход номер карты и возвращает ее маску.
- Функция, которая принимает на вход номер счета и возвращает его маску.
2. В модуле [Виджет](src\widget.py) находятся:
- Функция, которая принимает строку с данными карты или счёта и возвращает 
маску аккаунта карты банка или счёта непосредственного банка.
- Функция для отделения календарной даты от входящей строки с датой.
3. В модуле [Обработка](src\processing.py) находятся:
- Функция, которая создаёт новый список из заданного списка, фильтруя по заданному значению.
- Функция для сортировки данных пользователя по дате.
4. В модуле [Генераторы](src\generators.py) находятся:
- Функция, которая принимает на вход список словарей, представляющих транзакции.
Возвращает итератор, который поочередно выдает транзакции,
где валюта операции соответствует заданной (например, USD)
- Генератор, который принимает список словарей с транзакциями
и возвращает описание каждой операции по очереди.
- Генератор, который выдает номера банковских карт в формате: XXXX XXXX XXXX XXXX, где X— цифра номера карты.
Генератор должен принимать начальное и конечное значения для генерации диапазона номеров.
5. В модуле [Декораторы](src\decorators.py) находится:
- Декоратор для логирования выполнения функций, который будет автоматически регистрировать 
детали выполнения функций, такие как время вызова, имя функции, передаваемые аргументы, 
результат выполнения и информация об ошибках.\
Параметры:`filename (str):` путь к файлу для записи логов. При его отсутствии, результаты выводятся в консоль.
6. В модуле [Утилиты](src.utils.py) находится:
- Функция принимает ПУТЬ к файлу JSON и преобразует его в Python список, либо возвращает пустой Python список.
Пример пути к файлу: `'../data/operations.json'`
7. В модуле [Внешний API](src.external_api.py) находится:
- Функция для вывода суммы транзакции с конвертацией USD, EUR в RUB с использованием 
AIP сайта конвертации валют.
8. В модуле [Утилиты для csv и excel](src.csv_excel_utils.py) находятся:
- Функция, которая принимает ПУТЬ к файлу CSV и преобразует его в Python список словарей, 
либо возвращает пустой Python список.\
Пример пути к файлу: `'./data/transactions.csv'`
- Функция, которая принимает ПУТЬ к файлу CSV и преобразует его в Python список словарей, 
либо возвращает пустой Python список.\
Пример пути к файлу: `'./data/transactions_excel.xlsx'`
9. В модуле [Утилиты для поиска и подсчёта данных](src.re_utils.py) находятся:
- Функция для поиска заданной строки в списке словарей.
- Функция для подсчёта данных в списке словарей по заданным категориям.

В данный момент, для корректной проверки работы каждой функций, вы можете воспользоваться
оператором `print` и ввести все необходимые данные, соответствующие требуемым аргументам функций.

## Запуск и ожидаемое поведение программы
Для запуска программы воспользуйтесь:
```
python main.py
```
Программа находится на стадии альфа-теста. Для работы программы прилагаются данные транзакций в различных форматах
(JSON, CSV, xlsx), которые можно использовать в качестве тестирования работы функции.

Ожидаемое поведение программы должно быть следующим:

Программа приветствует пользователя:\

с банковскими транзакциями.

1. Получить информацию о транзакциях из JSON-файла
2. Получить информацию о транзакциях из CSV-файла
3. Получить информацию о транзакциях из XLSX-файла

Пользователь: 1\
Программа: Для обработки выбран JSON-файл.

Программа: Введите статус, по которому необходимо выполнить фильтрацию. 
Доступные для фильтровки статусы: EXECUTED, CANCELED, PENDING\
Пользователь: EXECUTED\
